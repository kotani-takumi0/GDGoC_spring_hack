# Implementation Plan

- [x] 1. プロジェクト基盤セットアップ
- [x] 1.1 Next.jsプロジェクトの初期化とベース設定
  - TypeScript、Tailwind CSS、App Routerを有効化したNext.jsプロジェクトを作成する
  - React Flow、dagre、react-syntax-highlighterの依存パッケージをインストールする
  - Gemini APIキーの環境変数（`GEMINI_API_KEY`）を`.env.local`で管理する設定を追加する
  - _Requirements: 7.2_

- [x] 1.2 シナリオ定義の静的JSONデータを作成する
  - Todoアプリシナリオの概念ノード（7個）、依存関係、フォールバックマッピングをJSONファイルに定義する
  - 経験レベル定義（complete-beginner / python-experienced / other-language-experienced）をJSONファイルに定義する
  - シナリオ一覧（Todoアプリのみavailable: true、他はcoming soon）を定義する
  - JSONデータを読み込むユーティリティ関数を実装する
  - _Requirements: 8.1, 8.2, 8.3, 8.4_

- [x] 1.3 (P) 共通レイアウトとナビゲーションを構築する
  - アプリ名「リアサポ」を表示するヘッダーを作成する
  - ステップインジケーター（1→2→3→4→5）を実装し、現在のステップを強調表示する
  - 白基調・青系アクセントカラーのデザイントーンを適用する
  - デスクトップ画面に最適化されたレイアウトを構築する
  - _Requirements: 6.1, 6.2, 6.3, 6.4_

- [x] 2. Gemini APIクライアントの構築
- [x] 2.1 Gemini API通信の共通基盤を実装する
  - サーバーサイドでGemini APIと通信する共通クライアントを作成する
  - 構造化出力モード（JSON mode + response_schema）による安定したJSONレスポンス取得を実装する
  - APIエラー時のリトライ（最大2回）とフォールバック値返却のエラーハンドリングを実装する
  - レスポンスのJSON parse失敗時の安全な処理を実装する
  - _Requirements: 7.1, 7.3, 7.4_

- [x] 2.2 (P) パーソナライズAPIエンドポイントを実装する
  - 経験レベルと概念ノード情報を受け取り、各ノードの一言説明をパーソナライズ生成するエンドポイントを作成する
  - Gemini APIへのプロンプトに経験レベルに応じた文脈を含め、初心者には平易な表現、経験者には技術的な比較を含む説明を生成する
  - 失敗時は静的JSONのデフォルト説明文をフォールバックとして返す
  - _Requirements: 2.2, 2.8_

- [x] 2.3 (P) コード生成APIエンドポイントを実装する
  - シナリオと経験レベルを受け取り、対応するコードを生成するエンドポイントを作成する
  - 生成されるコードがシナリオの全概念ノードをカバーするようプロンプトを設計する
  - 失敗時はエラーレスポンスを返し、クライアント側でリトライを促す
  - _Requirements: 3.1, 3.5_

- [x] 2.4 (P) マッピングAPIエンドポイントを実装する
  - 生成コードと概念ノード一覧を受け取り、各ノードに対応するコード箇所（行範囲・スニペット・説明）を判定するエンドポイントを作成する
  - 失敗時はシナリオ定義のフォールバックマッピングを返す
  - _Requirements: 4.1, 4.8_

- [x] 2.5 (P) 理解度評価APIエンドポイントを実装する
  - 概念ノード情報、対応コード、学習者のテキスト回答を受け取り、理解度（green/yellow/red）と一言フィードバックを返すエンドポイントを作成する
  - 判定基準：概念とコードの対応を正しく言語化できたか、方向性は合っているが曖昧か、的外れかの3段階
  - 失敗時はエラーレスポンスを返し、リトライを促す
  - _Requirements: 5.3, 5.4, 5.5, 5.9_

- [x] 3. Step 1 — シナリオ・経験レベル選択画面
- [x] 3.1 シナリオ選択と経験レベル選択の画面を構築する
  - シナリオ一覧をカード形式で表示し、利用可能なシナリオのみ選択可能にする
  - 未実装シナリオをグレーアウトし「Coming Soon」と表示する
  - 経験レベルの3つの選択肢をカード形式またはラジオボタンで表示する
  - 両方の選択が完了するまで「ロードマップを見る」ボタンを無効化する
  - 選択完了後、ボタンクリックでStep 2に遷移する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6_

- [x] 4. Step 2 — 概念ロードマップ表示画面
- [x] 4.1 React Flowで概念ロードマップを描画する
  - 静的JSONから取得した概念ノードと依存関係をReact Flowで表示する
  - カスタムノードとしてタイトル（技術用語）とサブテキスト（パーソナライズ説明）の2行を表示する
  - dagre自動レイアウトでノードを上から下に依存関係順で配置する
  - ノード間を矢印（曲線エッジ）で接続して依存関係を表現する
  - 全ノードを未評価状態（グレー統一色）で表示する
  - _Requirements: 2.3, 2.4, 2.5, 2.6_

- [x] 4.2 パーソナライズ説明の取得とロードマップ統合
  - 画面表示時にパーソナライズAPIを呼び出し、経験レベルに応じた説明文を取得する
  - API呼び出し中はローディング状態を表示する
  - 取得した説明文でノードのサブテキストを更新する
  - 「OK、実装して」ボタンを配置し、クリックでStep 3に遷移する
  - _Requirements: 2.1, 2.2, 2.7, 7.5_

- [x] 5. Step 3 — コード生成画面
- [x] 5.1 コード生成のローディング画面を構築する
  - 画面遷移と同時にコード生成APIを呼び出す
  - ローディングアニメーションと「AIが[シナリオ名]のコードを生成しています...」のテキストを表示する
  - Step 2のロードマップを薄く背景に表示し、待ち時間に概念を眺められるようにする
  - コード生成完了後、自動的にStep 4に遷移する
  - 生成失敗時はエラーメッセージとリトライボタンを表示する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 6. Step 4 — 概念↔コードマッピング表示画面
- [x] 6.1 左右パネルの並列レイアウトを構築する
  - 左パネル（幅40%）にロードマップ、右パネル（幅60%）にコードを配置する
  - 右パネルにシンタックスハイライト付きのコードビューアを実装する
  - 画面表示時にマッピングAPIを呼び出し、概念ノードとコード行範囲の対応を取得する
  - 取得したマッピングで各ノードの下にコードスニペットを追加表示する
  - _Requirements: 4.1, 4.2, 4.3_

- [x] 6.2 概念ノードとコードの接続線・インタラクションを実装する
  - 左パネルのノードと右パネルの対応コード箇所をカーブした接続線で結ぶ
  - ノードホバー時に該当する接続線と右パネルのコード箇所を強調表示する
  - ノードクリック時に右パネルの対応箇所にスクロールしてハイライトする
  - 「理解度チェックに進む」ボタンを配置し、クリックでStep 5に遷移する
  - _Requirements: 4.4, 4.5, 4.6, 4.7_

- [x] 7. Step 5 — 理解度評価画面
- [x] 7.1 回答フローとノード色変更を実装する
  - 左パネルにロードマップ（全ノードグレー状態）、右パネルに回答エリアを表示する
  - 最初の概念ノードを回答対象として選択し、対応コード箇所と質問「このコードが何をしているか、自分の言葉で説明してください」を表示する
  - テキスト入力エリアと「回答する」ボタンを配置する
  - 回答送信時に評価APIを呼び出し、判定結果（green/yellow/red）に基づいてノードの色を変更する
  - 一言フィードバックを回答エリアに表示する
  - 判定完了後、次のノードを自動的に回答対象に切り替える
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5, 5.6_

- [x] 7.2 進捗表示と完了画面を実装する
  - 回答の進捗（7問中○問回答済み等）を画面下部に表示する
  - 全ノードの回答完了後、ロードマップ全体が色づいた「自分の理解状態の地図」を完成形として表示する
  - _Requirements: 5.7, 5.8_

  - ローディング状態・エラー表示が各ステップで適切に動作することを確認する
