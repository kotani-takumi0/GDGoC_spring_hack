# ワークフロー図 生成プロンプト

以下のワークフローを、上から下に流れるフロー図として描いてください。

## レーン（横に3列）

左列: **ユーザー**（人間）
中央列: **AI（Gemini 3 on Antigravity）**
右列: **理解支援アプリ**（構造マップ）

## フロー

### ステップ①: 指示
- [ユーザー] → 「Todoアプリを作って」とAntigravityに入力
- 矢印 → [AI]

### ステップ②: タスク分解
- [AI] Gemini 3 がタスク分解を実行し、task.md を出力
  - Task 1: App.jsx（エントリポイント）
  - Task 2: TodoList.jsx（リスト表示）
  - Task 3: TodoItem.jsx（個別アイテム）
  - Task 4: useTodos.js（状態管理フック）
  - Task 5: storage.js（ローカルストレージ）
- MCP: `report_task_plan` → [理解支援アプリ]

### ステップ③: 構造マップの骨格描画
- [理解支援アプリ] task.mdをパースして:
  - 全ノードを⬜グレー（静止）で描画
  - 「依存」フィールドからエッジ（矢印）を描画
  - 「役割」フィールドからGemini Flashが事前学習の問題を生成

### ステップ④: 事前学習フェーズ（コードが書かれる前）
- [理解支援アプリ] → [ユーザー] に予想問題を出す
  - 例:「追加ボタンを押したとき、最初に動くファイルは？」
- [ユーザー] 回答する
- 正解のたびに構造マップ上のエッジがアニメーションで光る
- ※ この間、AIはまだ待機中 or Task 1 の実装を開始している

### ステップ⑤: AI実装開始
- [AI] Task 1（App.jsx）の実装を開始
- MCP: `report_progress`（ステップごとに複数回） → [理解支援アプリ]
- [理解支援アプリ]
  - App.jsxノードが ✨グレー点滅✨ に変化
  - 「AIの現在地」パネルに思考ステップをリアルタイム表示
    - 「useStateでTodoの配列を管理します」
    - 「forEachではなくmapを選択。理由: Reactでは戻り値が必要」

### ステップ⑥: ファイル実装完了
- [AI] App.jsx の実装完了
- MCP: `report_file_change` → [理解支援アプリ]
- [理解支援アプリ]
  - App.jsxノード: ✨グレー点滅 → ⬜白（静止）
  - Gemma 3（Cloud Functions）がコードを構造解析 → 内部ノード出現（useState, handleAdd, return JSX）
  - Gemini Flash（Vertex AI）が「壊して予想」問題を生成

### ステップ⑦: 壊して予想フェーズ
- [理解支援アプリ] → [ユーザー] に破壊シナリオの予想問題を出す
  - App.jsxノードが ✨白点滅✨（ユーザー取り組み中）
  - 例:「useStateを取り除くとアプリはどうなる？」
    - [A] Todoを追加しても画面に表示されない
    - [B] エラーで動かなくなる
    - [C] 見た目は変わらないがリロードで消える
- [ユーザー] 予想を選択
- Gemini Pro（Vertex AI）が回答を判定
- 結果:
  - 予想が当たった → 🟢緑
  - 惜しい → 🟡黄
  - 外れた → 🔴赤

### ステップ⑧: 🎉 Hello, World.
- [理解支援アプリ] 最初のノードが🟢緑に変わった瞬間、画面に「Hello, World.」と表示

### ステップ⑨: 並走の継続（⑤〜⑦の繰り返し）
- [AI] Task 2, 3, 4, 5... と順に実装を進める
- [ユーザー] AIが次のファイルを書いている間に、前のファイルの理解チェックを進める
- 構造マップ上で並走が可視化される:
  - AIが書いているノード → ✨グレー点滅
  - ユーザーが取り組んでいるノード → ✨白点滅
  - 理解済み → 🟢🟡🔴

### ステップ⑩: 全タスク完了
- [AI] 全ファイルの実装完了（点滅なし）
- [理解支援アプリ] 構造マップに🟢🟡🔴が混在
- 🔴赤ノード → NotebookLMが自動で音声解説を生成
  - ユーザーは他のノードの理解チェック中に耳で学べる

### ステップ⑪: 全ノード🟢緑 = プロジェクト全体を理解した状態

## ノード状態の凡例（図の端に配置）

| 表示 | 意味 |
|------|------|
| ⬜ グレー（静止） | task.mdに記載済み、未実装 |
| ✨ グレー（点滅） | AIが今まさにこのファイルを書いている |
| ⬜ 白（静止） | 実装済み、理解チェック待ち |
| ✨ 白（点滅） | ユーザーが理解チェックに取り組んでいる |
| 🟢 緑 | 予想が当たった（理解している） |
| 🟡 黄 | 惜しい（部分的に理解） |
| 🔴 赤 | 外れた（理解できていない） |

## MCP通信の凡例（図の端に配置）

| ツール名 | タイミング | 方向 |
|---------|-----------|------|
| `report_task_plan` | タスク分解直後（1回） | AI → アプリ |
| `report_progress` | AI実装中の各ステップ（複数回） | AI → アプリ |
| `report_file_change` | ファイル完成時（ファイルごとに1回） | AI → アプリ |

## Google技術の登場箇所（図中にラベルとして表示）

- ステップ②: Gemini 3（タスク分解・コード生成）
- ステップ③: Gemini Flash（事前学習の問題生成）
- ステップ⑤: Gemini 3 + report_progress
- ステップ⑥: Gemma 3（構造解析）、Gemini Flash（壊して予想の問題生成）
- ステップ⑦: Gemini Pro（回答判定）
- ステップ⑩: NotebookLM（音声解説）
- 全体基盤: Vertex AI、Firebase（Hosting / Cloud Functions / Firestore）

## 視覚的な要望
- 3レーンのスイムレーン図
- MCP通信は点線矢印で表現
- Google技術の登場箇所にはGoogleカラー（青・赤・黄・緑）のラベルを付ける
- ステップ⑤〜⑦が繰り返される（ループ）ことを矢印で示す
- ステップ⑧の「Hello, World.」は特別な装飾（星や光など）で強調
